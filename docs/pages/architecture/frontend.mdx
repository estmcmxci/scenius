## Split Layout

The `SplitLayout` component renders a draggable two-column layout:

- Default split: 65% left (paper), 35% right (chat)
- Drag handle between panels with `col-resize` cursor
- Constrained to 40–80% left width
- State managed via `leftPercent` with mouse event handlers

## Context System

`PaperReaderContext` is the central state bridge between the paper reader and chat:

```typescript
interface PaperReaderState {
  currentSection: string;
  selectedText: string;
  selectionRect: DOMRect | null;
  viewMode: 'markdown' | 'pdf';

  setCurrentSection(slug: string): void;
  setSelectedText(text: string, rect: DOMRect | null): void;
  setViewMode(mode: ViewMode): void;
  sendToChat(text: string): void;
  scrollToSection(slug: string): void;
  registerChatMethods(methods: ChatMethods): void;
  registerPaperScroll(fn: (slug: string) => void): void;
}
```

The chat panel registers its methods (`setComposerValue`, `focusComposer`, `sendUserMessage`) with the context on mount. The paper panel registers its scroll function. This lets either side trigger actions on the other without direct coupling.

## Custom Hooks

### `useIntersectionTracker`

Tracks which paper section is currently in the viewport using `IntersectionObserver`. Configured with a root margin of `-20% 0px -70% 0px` so sections are detected when they enter the upper portion of the scroll container.

## Dev Server

| Path | Target |
|------|--------|
